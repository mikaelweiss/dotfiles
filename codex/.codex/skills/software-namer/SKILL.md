# Software Namer

A skill for generating creative, memorable, and distinctive names for software projects.

## Core Philosophy

The best software names are **evocative but not literal**, **short but not truncated**, and
**distinctive but not alienating**. Think Stripe (simple, precise, metaphorical), Figma (invented,
soft, unique), Notion (conceptual, clean), Slack (familiar word, unexpected context), GitHub (literal
compound that works because it's playful).

**The goal is a name you'd be proud to write on a conference t-shirt** — not one that sounds like
it was generated by a machine or built by a committee.

---

## Step 1: Gather Information

Before generating names, ask the user these questions (combine into one message, not separate messages):

1. **What does the software do?** (brief description — 1–3 sentences)
2. **Who uses it?** (developers, designers, teams, consumers, enterprises, etc.)
3. **What vibe/feeling should the name evoke?** (e.g., precise/technical, friendly/approachable, powerful, minimal, playful, serious, fast)
4. **Any existing names they like or hate?** (helps calibrate taste)
5. **Any constraints?** (needs .com domain, must be one word, must be pronounceable in multiple languages, etc.)

If the user has already provided some of this, infer what you can and only ask what's missing.

---

## Step 2: Research Competitive Landscape

Before generating names, **use web search** to understand what names already exist in this space:

```
web_search: "[project type] software apps popular names"
web_search: "[key concept] app tool names"
```

Goals:
- Understand the naming conventions in this space
- Identify overused patterns to avoid
- Spot any names the user might accidentally collide with

---

## Step 3: Generate Names Using These Strategies

Apply **all** of these strategies, then filter to the strongest 8–12 names.

### Strategy 1: Real Word, Unexpected Context
Take a familiar English word and give it new meaning through context. The word should have some
conceptual resonance — a hint of meaning — but not be literal.

*Good examples:* Stripe (payments ≈ streamlined, precise), Slack (team chat ≈ looseness, ease),
Notion (notes ≈ concepts, ideas), Figma (design tool — invented but feels like "figure"), Linear
(project management ≈ directness), Vercel (deployment ≈ invented from "versatile")

*How to find these:* Think about the **feeling** or **metaphor** for the product. What does it
enable? What does the user feel when using it? Search for synonyms of those feelings.

### Strategy 2: Portmanteau (Word Blend)
Blend two relevant words. Aim for a blend that sounds like a real word, not a mashup.

*Good examples:* Pinterest (Pin + Interest), Instagram (Instant + Telegram), Netflix (Net + Flicks),
Snapchat (Snap + Chat), Brex (Brazil + Rex? — invented feel)

*How to do it:* List 5–8 words related to the project. Try overlapping them:
- Combine first half of word A + second half of word B
- Find where words share sounds or syllables

### Strategy 3: Invented/Abstract Word
Create a word that doesn't exist but sounds right. Should be pronounceable, memorable, and feel
like it could be a word.

*Good examples:* Figma, Kodak, Xerox, Kaggle, Heroku, Twilio, Trello

*How to do it:*
- Take a concept word and alter it slightly (change a vowel, drop/add a letter)
- Try unusual but pronounceable consonant + vowel combinations
- Aim for 2 syllables, ending in a vowel or soft consonant often works well

### Strategy 4: Metaphor / Conceptual Analogy
Find a metaphor from nature, mythology, architecture, or physical objects that captures what the
software does or enables.

*Good examples:* Amazon (vast, everything), Shopify (making commerce), Vercel (launching),
Dropbox (a box you drop things in), Basecamp (project management — a base camp before a climb),
Lighthouse (web auditing — shines light on problems)

*How to do it:* Ask: what is this tool LIKE? What physical or natural thing shares its essence?

### Strategy 5: Action/Verb Form
A gerund, verb, or action-oriented word that describes what users DO with the software.

*Good examples:* Buffer, Zapier, Render, Relay, Launch, Parse, Compile

*Note:* Works best when the action is slightly metaphorical, not literal.

### Strategy 6: Proper Noun Borrowed from Culture
A name borrowed from mythology, literature, geography, or history that has resonant meaning.

*Good examples:* Apollo (build tool — god of light and knowledge), Hermes (messaging — god of
communication), Atlas (maps, scale), Kafka (messaging queue — the labyrinthine bureaucracy
metaphor is apt), Prometheus (config management — bringing fire/config to the masses)

*Note:* Best for dev tools and infrastructure software. Less suitable for consumer apps.

---

## Step 4: Filter Against the Quality Checklist

For each name, score it mentally against these criteria. Only present names that pass most:

| Criterion | What to check |
|-----------|--------------|
| **Short** | Ideally 1–2 syllables; 3 is acceptable; 4+ is risky |
| **Pronounceable** | Can someone say it correctly on first read? |
| **Spellable from speech** | If someone hears the name, can they spell/search for it? |
| **Memorable** | Will it stick after one hearing? |
| **Distinctive** | Not similar to major existing software names |
| **Scalable** | Won't box in the product if it expands scope |
| **Domain-viable** | .com ideally; .io, .dev, .app acceptable for dev tools |
| **No clichés** | Does NOT use patterns from the avoid list below |

---

## Step 5: What to Avoid — The Cliché Blacklist

**NEVER suggest names that use these patterns:**

### Overused Suffix Patterns (stale, impersonal)
- `-ly` suffix: Speakly, Dictly, Writly
- `-r` vowel drop: Speakr, Writr, Flickr-style (2007 called)
- `-ify`: Speechify, Taskify (Spotify already owns this sound territory)
- `-hub`: TaskHub, CodeHub, DataHub
- `-stack`: DevStack, AppStack
- `-kit`: VoiceKit, DataKit, UIKit
- `-pro`: TaskPro, SyncPro, WritePro
- `-ai` tacked on: SpeakAI, WriteAI, TaskAI
- `-io` as part of the name (not domain): Taskio, Notesio

### Generic Tech Buzzword Combos (meaningless, corporate-sounding)
- Words: Smart, Ultra, Hyper, Turbo, Meta, Nexus, Apex, Prism, Core, Edge, Flux, Nova, Pulse
- Combos like: SmartTask, HyperSync, UltraNote, CoreFlow, EdgeKit
- Anything with: Cloud, Digital, Virtual, Tech, Data, Net (as part of the name, not metaphorically)

### Bland Descriptive Names (no personality)
- "[What it does] + App/Tool/Manager/Suite/Platform"
- TaskManager, FileSync, VoiceRecorder, CodeEditor

### AI-Era Clichés (overused as of 2024–2025)
- Anything ending in `-GPT`, `-AI`, `-Bot`, `-LM`
- "Jarvis-adjacent" names implying an AI assistant
- Names that lean on "intelligence", "brain", "neuro", "cogn-"

---

## Step 6: Present Names with Context

For each suggested name, provide:

1. **The name** (bold)
2. **Pronunciation** (if non-obvious)
3. **Why it works** — 1–2 sentences on the resonance/strategy used
4. **The conceptual hook** — what feeling or idea it connects to
5. **Domain note** — brief note on likely availability

Present in groups by strategy type, 2–4 names per strategy. Aim for **8–12 total names**.

Example format:
> **Quill** — *Real word, unexpected context*
> A quill is a writing instrument — ancient, precise, deliberate. For a note-taking app, it implies
> craft and intentionality. The word is short, distinctive, and has the hard-Q energy that reads
> as modern. `.com` likely taken; `quill.app` or `useQuill.com` are options.

---

## Step 7: Help the User Narrow Down

After presenting names, ask:
- Which names resonate, and why?
- Any that feel close but not quite right? (often leads to the best names through iteration)

Offer to:
- Generate variations on names they like
- Combine elements from multiple candidates
- Adjust the vibe (more serious, more playful, etc.)
- Do a quick web search to check if a specific name is already taken

---

## Notes on Domain & Availability

When a user seems close to a final choice, suggest they check:
- **Namecheap or GoDaddy** for domain availability
- **USPTO TESS** (or equivalent in their country) for trademark conflicts
- **GitHub, npm, PyPI** (etc.) if it's a dev tool/library — package name squatting is real
- Social handles: the name should ideally be available on at least one major platform

.com is best, but `.io`, `.dev`, `.app`, `.so` are all credible for software products in 2025+.
Avoid `.net`, `.org` for commercial software — they imply legacy or non-profit.

---

## Special Case: Open Source Libraries & Dev Tools

For packages, CLIs, and libraries (not end-user products), the naming calculus shifts:

- **Brevity is even more important** — it'll be typed constantly (`npm install quill`, `pip install quill`)
- **Mythology and culture references work great** here (developers appreciate the nod)
- **Avoid names already taken on npm/PyPI/crates.io** — search before suggesting
- Alliteration and internal rhyme help (`ripgrep`, `tokio`, `hyper`, `axum`)
- It's OK to be slightly more abstract or cryptic — developers will look it up

For CLI tools specifically: short, punchy names that are easy to type win. Think `grep`, `curl`, `wget`, `jq`, `fzf`.
